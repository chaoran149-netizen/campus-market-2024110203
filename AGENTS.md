# 语言与交互偏好

- 始终使用**简体中文**回复用户
- 代码注释使用中文
- 文档内容使用中文
- 项目输出（证据卡、README 等）使用中文
- 技术术语可以保留英文，但解释用中文

# 证据卡（Evidence）写作规范

当用户要求填写 Evidence 文件（如 `docs/evidence/Day*_Evidence.md`）时，必须遵守以下流程：

## 强制四步流程

1. **抽取模板** — 如果用户提供了模板/示例结构，先把章节标题逐条抽成清单，不是扫一眼就过
2. **逐章对齐** — 按清单逐章写作，模板用表格就写表格，模板要操作步骤+预期+结果就补三列
3. **自检勾选** — 写完后对照模板逐项检查，每一章都勾过才提交
4. **禁止偷懒** — 不允许只列要点、跳过章节、用短段落代替完整结构

## 证据卡最少包含项

- Store 设计说明（表格形式）
- 状态边界说明（表格形式：哪些数据入 Store、哪些不入、原因）
- 页面使用记录（表格形式：页面名 + Store 名 + 使用方式）
- AI 协作记录（工具 + 提示词 + 生成内容 + 潜在问题）
- 人工调整内容（逐项列出）
- 测试记录（表格形式：测试项 + 操作步骤 + 预期结果 + 实际结果）
- 遇到的问题与解决方法（至少一个真实问题）
- 今日反思（一小段话总结收获）

## 禁止行为

- 只粘贴代码不写设计说明
- 把 AI 生成的完整登录系统/Auth 逻辑直接作为成果
- 证据卡少于 300 字
