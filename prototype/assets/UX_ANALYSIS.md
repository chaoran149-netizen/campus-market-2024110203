# 校园轻集市 — UX 分析与产品规划

## 一、用户角色分析

| 角色 | 说明 |
|------|------|
| **普通学生** | 浏览信息、搜索筛选、查看详情、收藏内容、发送消息 |
| **信息发布者** | 发布四类信息、管理自己的发布、更新状态 |
| **信息浏览者** | 主动搜索筛选、浏览信息列表、查看详情、联系发布者 |
| **消息沟通双方** | 通过消息中心进行会话交流、查看聊天记录 |
| **平台管理者** | 审核内容、查看数据看板、处理违规举报 |

## 二、核心业务场景

| 场景 | 用户行为 |
|------|----------|
| 二手交易 | 发布闲置 → 浏览商品 → 查看详情 → 收藏/联系卖家 |
| 失物招领 | 发布失物/拾物 → 浏览招领 → 查看详情 → 联系发布者 |
| 拼单搭子 | 发布拼单 → 浏览拼单信息 → 加入拼单 → 查看人数/截止时间 |
| 跑腿委托 | 发布委托 → 浏览任务 → 查看报酬 → 接单联系 |
| 消息沟通 | 查看会话列表 → 发送消息 → 接收回复 |
| 个人中心 | 查看/编辑资料 → 管理发布 → 查看收藏 → 账号设置 |
| 管理端 | 数据概览 → 内容审核 → 违规处理 |

## 三、核心交互流程

```
浏览流程：  首页 → 分类浏览 → 搜索/筛选 → 查看详情
发布流程：  首页 → 发布选择 → 填写表单 → 提交成功
沟通流程：  详情页 → 联系发布者 → 发送消息 → 接收回复
收藏流程：  详情页/列表 → 点击收藏 → 个人中心查看
管理流程：  管理首页 → 审核列表 → 通过/驳回 → 处理违规
```

## 四、页面规划

| 页面 | 功能目标 | 关键内容 |
|------|----------|----------|
| 首页 | 引导进入各业务 | 快捷入口、最新信息、数据概览 |
| 分类浏览 | 信息检索与筛选 | 搜索框、筛选器、信息卡片、分页 |
| 商品详情 | 展示完整信息 | 图片、描述、价格、联系、收藏 |
| 失物详情 | 展示失物信息 | 物品特征、时间地点、联系 |
| 拼单详情 | 展示拼单信息 | 人数进度、截止时间、参与 |
| 任务详情 | 展示任务信息 | 酬劳、地点、时间、联系 |
| 发布选择 | 选择发布类型 | 四类业务入口卡片 |
| 各发布页 | 填写表单发布 | 动态表单、图片上传、须知 |
| 消息列表 | 管理会话 | 会话列表、未读标记、搜索 |
| 聊天详情 | 实时沟通 | 聊天记录、发送、常用回复 |
| 个人中心 | 用户管理 | 资料、发布、收藏、设置 |
| 管理端 | 平台管理 | 数据看板、审核表格、快捷操作 |
