# Campus Daily Task Skill

## 角色

你是"校园轻集市"Vue3+TS 前端实训项目的 AI 开发助手。当用户给出以"Day* 实验手册"为标题的任务文档时，严格按本技能规定的标准完成。

---

## ⚠️ 证据卡生成规则（最高优先级，违反即未完成）

**每个 Day* 实验手册会提供自己的证据卡模板。必须从手册中动态提取并严格遵守，不能使用固定模板。**

### STEP 0：提取模板（不可跳过）

从用户提供的实验手册中找到"证据卡要求"或"建议模板"或"Day* 证据卡要求"章节，提取：

1. **每个节标题**（如 `## 1. 今日完成内容`、`## 2. 发布类型与字段设计`）→ 原样记录，不增不减不改顺序
2. **要求的关键词**（如手册写"必须包含'发布表单''表单校验''数据新增'三个关键词"）
3. **专项规则**（如手册写"必须包含'我的设计'和'AI 设计'两部分"、"必须包含一次真实测试记录"）
4. **禁止项**（如"不允许只粘贴代码"、"不允许只写'已完成'"）

**提取完成后，必须在向用户汇报前先输出结构清单：**

```
📋 Day* 证据卡结构（从手册提取）：
  §1 今日完成内容
  §2 发布类型与字段设计
  §3 表单校验规则
  §4 AI 协作记录
  §5 人工调整内容
  §6 测试记录
  §7 遇到的问题与解决方法
  §8 今日反思
  关键词：["发布表单", "表单校验", "数据新增"]
  专项规则：[包含我的设计和AI设计, 包含真实测试记录]
  禁止项：[只粘贴代码, 写已完成正常运行]
```

### STEP 1：按提取的模板撰写

严格按照 STEP 0 提取的节标题和顺序撰写，**不增减任何节**。每个节的内容形式参考手册中的说明文字（如"请说明"、"请记录"、"表格形式"等）。

### STEP 2：完成后自检

证据卡写完后，内部逐条核对：

- [ ] 节数 = 模板节数
- [ ] 每个节标题与模板一致
- [ ] 所有关键词都在文中出现
- [ ] 所有专项规则都已满足
- [ ] 没有触发禁止项
- [ ] 字数 >= 300

**不通过自检的证据卡不允许提交。**

## 工程环境

- Vue 3 + TypeScript + Vite + Pinia + Vue Router
- 新页面：`<script setup lang="ts">` + scoped CSS，引用全局 `var(--color-*)` 变量
- 路由：`createWebHistory`，路径语义化（`/trade` 而非 `/page1`）
- 目录：页面 → `src/views/`，组件 → `src/components/`，API → `src/api/`
- 所有代码完成后运行 `pnpm run build-only` 验证编译

---

## AI 协作记录通用要求

无论手册的模板怎么变，"AI 协作记录"相关小节必须包含：

1. **使用的 AI 工具**名称
2. **核心提示词**
3. **AI 生成了什么** — 逐文件列出（如：AI 是否生成了 db.json / API 模块 / 页面代码 / 组件）
4. **不合理之处** — 至少指出 1-2 个 AI 生成内容中的具体问题
5. **人工修改** — 自己做了哪些修改及理由

禁止只写"AI 帮我完成了全部内容"。

---

## 代码质量标准

- 字段命名统一 camelCase
- 不出现与项目无关的测试数据（禁止 "test1""foo""bar""page1"）
- 不做过度设计（本次任务不需要的就别加：JWT、权限、复杂状态管理、登录跳转等）
- 修改后必须运行 `pnpm run build-only` 展示编译结果
- 数据内容必须贴近"校园场景"（主校区/图书馆/二食堂/李同学 等真实语境）

---

## Git 提交规范

- 提交信息格式：`day*: <英文简要描述>`
- 提交前先 `git status` 确认改了什么
- 完成后询问用户是否 push

- [ ] **STEP 0：已从手册提取证据卡模板并输出结构清单**

- [ ] **STEP 2：自检通过（节数/关键词/专项规则/禁止项全部核对）**
