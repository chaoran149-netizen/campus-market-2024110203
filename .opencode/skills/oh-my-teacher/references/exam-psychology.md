# Exam Psychology and Performance Routines

用于 `/exam-routine`、`/mock` 考前训练、`/cram` 焦虑管理，以及考试临场策略教学。这是考试表现训练，不是心理治疗。

## Exam Routine (考前流程)

### 考前 10 分钟

```markdown
## 考前 10 分钟流程
1. 快速浏览全卷，标记每题预估难度（⭐简单 / ⭐⭐中等 / ⭐⭐⭐困难）
2. 按"先易后难"排出答题顺序
3. 为每部分分配时间预算（按分值比例）
4. 设一个"检查时间点"：考试时间的 85% 处停笔检查
5. 深呼吸三次，从最有把握的题开始
```

### 卡题恢复脚本 (Stuck Recovery)

当学生在模考或练习中卡住超过合理时间：

```markdown
## 卡题恢复（60 秒）
1. 标记题号，先写出已知条件和能拿的步骤分
2. 30 秒内判断题型：定义 / 公式 / 模板 / 证明 / 计算 / 分析
3. 若仍无思路：
   - 证明/推导题：写出定义和已知条件，至少拿"开头分"
   - 计算题：列出公式和代入步骤，至少拿"过程分"
   - 论述题：写出核心概念名词和框架，至少拿"要点分"
4. 果断跳到下一题
5. 最后回看时只检查：单位、条件、符号、边界、答案格式
```

### 交卷前检查清单

```markdown
## 交卷前 5 分钟
- [ ] 选择题有没有漏涂/涂错
- [ ] 计算题有没有写单位和结论
- [ ] 证明题有没有写"综上所述"或结论
- [ ] 编程题有没有检查边界输入
- [ ] 论述题有没有扣回题目关键词
- [ ] 个人信息有没有填写
```

## Anxiety Reappraisal (焦虑重评)

当学生触发 `/cram` 或使用高焦虑词汇（"完蛋了""必挂""来不及了""怎么办"）时，在给出复习计划前插入一句焦虑重评：

- 原则：将焦虑重构为"战斗准备状态"，不是劝学生"别紧张"。
- 示例：
  - "心跳加速说明你的大脑正在全速运转准备战斗。来，先干掉这道最简单的题。"
  - "紧张是正常的考试反应，说明你在认真对待。我们先花 5 分钟拿下一个 quick win。"

不要：
- 做心理治疗或深度情绪疏导
- 说"别紧张""放轻松"（这些会增加焦虑）
- 花超过 2 句话在情绪管理上——立刻转入具体行动

## Overconfidence Warning (过度自信预警)

当 confidence calibration 发现 confidence_gap >= 2 的主题时：

```markdown
⚠️ 过度自信预警：[topic] 你预测把握度 [X]/5，但实际表现 [Y]/[max]。
这类"感觉会了但考试会丢分"的盲区最危险。建议立即用 /fix 或 /quiz 验证。
```

在 `/dashboard` 和 `/plan` 中，高信心错题始终排在复习优先级最前。

## Mock Exam Psychology Integration

在 `/mock` 流程中嵌入考试心理训练：

1. **考前**：生成模考卷后，先进行审题分流训练（见 `references/practice-workflows.md` → Triage Training）。
2. **考中**：如果学生报告卡住，给出卡题恢复脚本。
3. **考后**：模考复盘中加入心理复盘：
   - 哪题因为紧张而表现异常？
   - 时间分配是否合理？
   - 有没有"会做但没时间做"的遗憾？

## Sleep and Energy (睡眠与精力提醒)

在 `/plan` 生成多日计划时，加入轻量提醒：

- 考前一天不安排新难点，只做已学内容的快速回顾和考前一页纸。
- 高负荷的抽象概念学习（定理证明、新算法）排在上午或精力充沛时段。
- 睡前 1 小时适合做 `/review-due` 旧概念回忆，利用睡眠巩固。
- 不鼓励通宵：疲劳状态下的学习效率极低，且会损害考试当天的工作记忆。

## Scope

这个参考文件的定位是**考试表现训练**（performance routine），不是心理咨询。只提供基于认知科学的临场策略和轻量情绪管理。不做诊断、不做治疗、不替代专业心理支持。如果学生表现出严重焦虑或心理困扰，建议联系学校心理咨询中心。
