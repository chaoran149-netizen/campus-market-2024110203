## 4. 代码演示 `/code-demo`

**用户**：能给我讲讲 BFS 为什么出错吗？

**Skill**：

> **概念回顾**
>
> BFS（广度优先搜索）用队列维护待访问顶点，逐层遍历图。关键是队列的**入队 / 出队顺序**。
>
> **演示代码（Python）**
>
> ```python
> from collections import deque
>
> def bfs(graph, start):
>     visited = set([start])
>     queue = deque([start])
>     result = []
>     
>     while queue:
>         node = queue.popleft()  # 关键：从队首出队
>         result.append(node)
>         
>         # 遍历邻接点，未访问的入队
>         for neighbor in graph[node]:
>             if neighbor not in visited:
>                 visited.add(neighbor)
>                 queue.append(neighbor)  # 从队尾入队
>     
>     return result
>
> # 测试
> graph = {
>     0: [1, 2],
>     1: [0, 3],
>     2: [0, 3],
>     3: [1, 2]
> }
> print(bfs(graph, 0))  # 输出：[0, 1, 2, 3]
> ```
>
> **执行追踪**
>
> ```
> 初始状态：queue = [0], visited = {0}
>
> 第 1 步：
>   - 出队 0，result = [0]
>   - 邻接点 1, 2 都未访问，入队
>   - queue = [1, 2], visited = {0, 1, 2}
>
> 第 2 步：
>   - 出队 1，result = [0, 1]
>   - 邻接点 0 已访问，3 未访问，入队 3
>   - queue = [2, 3], visited = {0, 1, 2, 3}
>
> 第 3 步：
>   - 出队 2，result = [0, 1, 2]
>   - 邻接点 0, 3 都已访问，无需入队
>   - queue = [3]
>
> 第 4 步：
>   - 出队 3，result = [0, 1, 2, 3]
>   - queue 为空，终止
> ```
>
> **常见错误**
> - ❌ 使用 `queue.pop()` 而不是 `popleft()` → 变成 DFS
> - ❌ 入队后不标记 visited → 队列重复增长，无限循环
> - ❌ 出队后才标记 visited → 同一节点入队多次
>
> **练习**：修改代码为邻接矩阵输入，试试。

## 7. 错题修复 `/fix`

**用户**：T4 动态规划题我没做对。

**Skill**：

> **微课堂：0-1 背包状态转移**
>
> 核心：`dp[i][w]` = 前 i 个物品，容量 w 时的最大价值。
>
> 转移方程：
> ```
> dp[i][w] = max(
>     dp[i-1][w],                              // 不选物品 i
>     dp[i-1][w-weight[i]] + value[i]         // 选物品 i
> )
> ```
>
> 前提：`w >= weight[i]`
>
> **模板代码**
>
> ```python
> def knapsack(n, W, weights, values):
>     # dp[w] = 容量 w 的最大价值（滚动数组）
>     dp = [0] * (W + 1)
>     
>     for i in range(n):
>         # 从后往前遍历容量，避免一个物品用两次
>         for w in range(W, weights[i] - 1, -1):
>             dp[w] = max(dp[w], dp[w - weights[i]] + values[i])
>     
>     return dp[W]
> ```
>
> **常见错误**
> - ❌ 从前往后遍历容量 → 同一物品被用多次（变成完全背包）
> - ❌ 初始化为 INF 或负数 → 比较时崩溃或错误
> - ❌ 混淆 0-1 和完全背包 → 复杂度或结果错误
>
> **变式练习**
> - 基础：经典 0-1 背包
> - 变式：求是否恰好装满；求方案数
> - 综合：分组背包（多类物品，每类选至多一个）
